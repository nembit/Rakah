Privacy Policy

Rakah — Islamic Prayer Tracker  ·  Effective April 16, 2026

🔒 Rakah collects absolutely no personal data.

Everything you enter — prayer logs, qada counts, settings — lives exclusively on your device. It is never transmitted to any server or third party.

1. Overview

Rakah ("the app", "we", "us") is a free, offline-first Islamic prayer tracking application for Android, iOS, and Web. This Privacy Policy explains what information the app accesses, how it is used, and your rights as a user.

Because Rakah is local-first by design, the vast majority of this document confirms what we do not do.

2. Data Collected & Stored

All data is stored locally on your device using AsyncStorage (React Native's standard on-device key-value store). No data is synced or backed up to a cloud service operated by Rakah.

3. Device Permissions

Rakah requests the following device permissions. Each permission is optional unless stated otherwise, and is used solely for the feature described.

  • Location (foreground only) — Used to automatically detect your coordinates for accurate prayer times and Qibla direction. Your location is read once per session and stored on-device as latitude/longitude. It is never sent to any server.
  • Notifications — Used to send local, on-device prayer time reminders scheduled by the app. No server is involved in delivering these notifications.
  • Magnetometer & Accelerometer — Used by the Qibla compass screen to determine the device's heading. Sensor data is processed in real time and never stored or transmitted.
  • Microphone (RECORD_AUDIO / MODIFY_AUDIO_SETTINGS) — Declared as required by an underlying audio SDK. Rakah does not actively use the microphone for any feature and does not record audio.

4. Data Export & Import

Rakah provides a manual data export feature in Settings that writes all your prayer data to a JSON file on your device. You control what you do with that file. The app never automatically uploads backups anywhere.

You can also import a previously exported file, or clear all local data at any time from Settings → Clear All Data.

5. Third-Party Services

Rakah uses only a small number of third-party services, none of which receive personal or prayer data:

  • Nominatim / OpenStreetMap — Used when you search for a city name in the location picker. The search query (a city name you type) is sent to the public Nominatim API to return coordinates. No personally identifiable information accompanies this request. See Nominatim's usage policy.
  • TimeZoneDB — Optionally used to look up the UTC offset for a selected city. Only the latitude/longitude of the selected location is sent. See TimeZoneDB's privacy policy.
  • Expo (EAS) — Used to build and distribute the app. Expo processes metadata during the build process but does not have access to user data inside the app. See Expo's privacy policy.

Rakah does not use analytics SDKs, crash reporting services, advertising SDKs, or any data broker services.

6. Children's Privacy

Rakah does not knowingly collect any information from anyone, including children under the age of 13. Because no data is collected or transmitted, the app is suitable for users of all ages.

7. Data Security

Because all data remains on your device, its security depends on your device's own security (lock screen, encryption, etc.). We recommend keeping your operating system and the app updated. If you lose your device or uninstall the app, your data is not recoverable unless you have created a local backup using the export feature.

8. Changes to This Policy

If we ever change how Rakah handles data, we will update this page and revise the effective date at the top. Significant changes will also be noted in the app's release notes.

9. Contact

If you have questions about this privacy policy, please reach out at support@rakah.app.
